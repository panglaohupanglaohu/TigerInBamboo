## 彻底移除 three import map 依赖（修复 "blocked by a null value"）
- 根因：运行时（IDE 预览/包装层）注入含 `"three": null` 的第二个 import map，屏蔽页面自带 map。
- 方案：全部改相对路径导入，删除所有 import map。
  - `frontend/js/*.js`(18) 与 `frontend/js/bio/*.js`(5)：`three`→`../assets/vendor/three/three.module.js`(bio 用 `../../`)；`three/addons/`→`.../jsm/`；`cannon-es`→`../assets/vendor/cannon-es.js`。
  - three addon 内部 3 文件(OrbitControls/GLTFLoader/BufferGeometryUtils)：`from 'three'`→`../../three.module.js`。
  - `index.html`/`plum.html`/`lab.html` 删除 `<script type="importmap">` 块。
- 项目约定：three r160 本地 vendored 于 `frontend/assets/vendor/three/`(three.module.js + jsm/)，cannon-es 于 `frontend/assets/vendor/cannon-es.js`；今后新增模块一律用相对路径导入，勿再引入 import map。
